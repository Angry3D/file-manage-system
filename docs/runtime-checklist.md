# 真实运行 Smoke Checklist

最后更新：2026-05-31  
语系：cn

本文档用于 TODO-0005 第二批（Todo 4）手工验证，不替代自动化验证脚本。

## 1. 本地运行前置条件

1. Node.js：满足当前仓库约束（建议与 `server/package.json` 的 `engines.node` 保持一致，至少 `>=14.15.0`）。
2. pnpm：满足根 `package.json` 的 `engines.pnpm`（`>=10 <11`）。
3. MySQL：可连接，且已准备项目运行所需数据库与表结构（仓库未提供 schema，需外部准备）。
4. 环境变量 `BABYLIFE_DB_PASSWORD`：已在本地注入正确数据库密码。
5. 上传目录：
   - 开发环境默认 `/Users/relax/Documents/upload`；
   - 或通过 `BABYLIFE_UPLOAD_DIR` 覆盖到本机可读写目录。
6. Sharp 构建依赖：已执行 `pnpm install`，如本机存在 native 包构建拦截，已按提示执行 `pnpm approve-builds`。

## 2. 推荐启动顺序

1. 启动服务端（API）：
   - 在仓库根执行 `pnpm --filter server start`（或进入 `server/` 执行 `npm start`）。
2. 启动文件服务（静态上传目录）：
   - `server/script/file-server.sh` 会在 `npm start` 中被调用；
   - 如需单独启动，确认端口与上传目录和服务端配置一致。
3. 启动管理端（web-admin）：
   - 在仓库根执行 `pnpm --filter web-front serve`（或进入 `web-admin/` 执行 `npm run serve`）。
4. 启动 H5（web-h5）：
   - 在仓库根执行 `pnpm --filter web-h5 serve`（或进入 `web-h5/` 执行 `npm run serve`）。

## 3. 手工 Smoke Test 步骤与预期结果

### 3.1 登录（管理端）

操作：
1. 打开管理端登录页。
2. 输入有效账号密码并登录。

预期结果：
1. 登录成功后跳转到管理端主界面。
2. 浏览器中存在 `adminToken`（或等效 token 持久化项）。
3. 后续管理端请求可正常携带 token。

### 3.2 上传 JPG/PNG/GIF（管理端图片新增页）

操作：
1. 进入图片新增页。
2. 分别上传一张 `JPG`、一张 `PNG`、一张 `GIF`。

预期结果：
1. 三种格式都可被前端接受并进入上传流程。
2. 上传成功后可看到对应缩略图预览。
3. 任一上传失败时，有可理解错误提示，失败项不会被当作可提交图片。

### 3.3 新增图片

操作：
1. 在图片新增页选择已上传完成的图片。
2. 填写/不填写地点与备注（按业务允许范围）。
3. 点击“确定”提交新增。

预期结果：
1. 提交成功时提示“添加成功”，并返回列表页。
2. 提交失败时按钮 loading 能恢复，页面可继续操作。
3. 新增记录可在管理端列表查询到。

### 3.4 编辑图片

操作：
1. 从列表进入某条图片记录的编辑页。
2. 可选替换图片，调整状态/地点/备注。
3. 点击“确定”提交编辑。

预期结果：
1. 提交成功时提示“编辑成功”，并返回列表页。
2. 提交失败时按钮 loading 能恢复，页面可继续操作。
3. 返回列表后可看到最新编辑结果（图片、状态、地点、备注）。

### 3.5 删除图片

操作：
1. 在管理端列表选择一条或多条记录执行删除。

预期结果：
1. 删除操作成功后列表刷新，记录不再显示。
2. 删除行为仅验证业务记录层；本轮不验证物理文件清理策略。

### 3.6 H5 展示

操作：
1. 打开 H5 照片墙页面。
2. 验证已展示状态图片是否可见。
3. 点击图片进入预览。
4. 人工模拟空列表、错误图片地址、接口失败等场景（可通过测试数据或临时代理）。

预期结果：
1. 正常数据下图片可加载、可预览。
2. 空列表时显示明确空状态文案。
3. 图片加载失败时显示兜底占位，不出现破图。
4. 请求失败后 loading 会恢复，并有轻量提示或可理解状态。

## 4. 失败排查入口

1. 服务端日志：重点查看 `server` 控制台输出和上传处理异常。
2. 文件服务：检查上传目录读写权限、端口占用、静态服务是否可访问。
3. 管理端/H5 前端日志：浏览器控制台网络请求与报错信息。
4. 环境变量：核对 `BABYLIFE_DB_PASSWORD`、`BABYLIFE_UPLOAD_DIR`、`BABYLIFE_FILE_HOST`。

## 5. 已知未覆盖项

1. 仓库未提供 SQL schema / migration，无法在仓库内独立重建完整数据库结构。
2. 初始管理员账号信息未在仓库中提供，需要外部提供或本地已有环境。
3. 真实上传链路自动化测试（含 Sharp、文件系统、数据库联动）目前未落地，仍以手工 smoke 为主。
