## 6. Todo 清单

### Todo 1：修复管理端图片保存错误处理

- 修改位置：`web-admin/src/views/file/image/ImageAdd.vue`
- 修改内容：
  - 修复编辑模式中 `this.$$Message.error` 为 `this.$Message.error`。
  - 为新增和编辑提交补充失败后的 `okBtnLoading = false`。
  - 避免接口失败后按钮一直处于 loading 状态。
- 修改原因：该拼写错误真实存在；保存请求失败时 loading 不复位会影响管理端继续操作。
- 预期结果：新增/编辑失败后用户能看到错误提示或至少恢复按钮状态。
- 完成判定：代码中不再出现 `this.$$Message`；新增/编辑 Promise 失败路径会恢复 `okBtnLoading`。
- 验证方式：`pnpm run verify:web-admin`；必要时手工模拟接口失败。
- 注意事项：不要改变表单字段结构和接口路径。
- 依赖项：无。
- 是否阻塞：否。
- 难度等级：简单。
- 推荐执行者：普通执行 AI。
- 风险说明：低风险，局部前端修复。

### Todo 2：补强服务端图片新增/编辑参数校验

- 修改位置：`server/src/logic/admin/image.js`
- 修改内容：
  - `addAction` 增加对 `images` 数组元素的字段校验，至少要求 `url`、`url_thumb`、`id` 存在。
  - `setAction` 增加 `image_thumb`、`image_id` 校验，使 logic 与 controller 写入字段一致。
  - 必要时限制 `status` 只允许展示/隐藏状态。
- 修改原因：当前控制器依赖这些字段写入数据库，但 logic 没有完整校验，可能写入 undefined 或非法状态。
- 预期结果：非法图片参数会在 logic 层返回参数错误，不进入数据库写入。
- 完成判定：缺少图片 URL、缩略图或上传 ID 的请求不会通过 logic 校验。
- 验证方式：`pnpm run verify:server`；必要时补最小逻辑验证或手工请求验证。
- 注意事项：不要引入数据库 schema 假设；不要改变成功响应格式。
- 依赖项：无。
- 是否阻塞：否。
- 难度等级：中等。
- 推荐执行者：普通执行 AI。
- 风险说明：可能让历史上字段不完整的调用失败，但这是期望的质量收紧。

### Todo 3：补强上传组件失败处理

- 修改位置：`web-admin/src/components/XUpload.vue`
- 修改内容：
  - 增加上传失败事件处理，例如网络错误或服务端非业务成功响应。
  - 对非 201/202 但 `errno` 非 0 的响应给出通用错误提示。
  - 确认失败文件不会被误认为已完成可提交。
- 修改原因：当前仅处理 201/202，网络失败或其他错误码时反馈不足。
- 预期结果：上传失败时用户能看到明确提示，保存页不会把失败项当成已完成图片。
- 完成判定：上传失败路径有用户提示；`file.url`、`file.urlThumb`、`file.urlID` 只在成功数据完整时写入。
- 验证方式：`pnpm run verify:web-admin`；手工模拟上传接口失败更佳。
- 注意事项：保持 View Design Upload 的既有用法，不替换上传组件库。
- 依赖项：无。
- 是否阻塞：否。
- 难度等级：中等。
- 推荐执行者：普通执行 AI。
- 风险说明：需要理解 Upload 组件事件，但影响范围局部。
